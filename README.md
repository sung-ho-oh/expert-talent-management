# 🚀 Google Apps Script 기반 전문가 인재풀 관리 시스템

본 프로젝트는 **Google Workspace(Sheets, Drive)** 환경을 활용하여 서버 비용 없이 독립적으로 운영 가능한 전문가 데이터 관리 솔루션입니다. Google Apps Script(GAS)의 웹 앱 호스팅 기능을 통해 프론트엔드와 백엔드를 통합 구현하였습니다.

## 🌟 핵심 기술적 차별점 (Learning Points)

### 1. 이미지 보안 정책(CSP) 우회 전략

Google Drive에 저장된 이미지를 웹 브라우저에서 직접 호출할 경우, 구글의 강화된 보안 정책(CORS/SameSite Cookie)으로 인해 이미지가 깨지는 현상이 빈번합니다.

* **해결 방법**: 클라이언트가 URL로 직접 이미지를 요청하는 대신, **서버(`Code.gs`)가 드라이브 파일을 바이너리로 읽어 Base64 텍스트 데이터로 변환**하여 전송합니다.
* **효과**: 브라우저 설정이나 구글 로그인 상태와 관계없이 100% 안정적인 이미지 렌더링을 보장합니다.

### 2. 매니페스트를 통한 인프라 설정 (IaC)

협업 시 발생하는 권한 승인 오류를 방지하기 위해 `appsscript.json`에 필요한 권한 범위를 명시적으로 정의했습니다.

* **설계**: `oauthScopes` 선언을 통해 프로젝트가 사용하는 리소스를 투명하게 공개하고 승인 절차를 간소화했습니다.

### 3. 고정형 네비게이션 UI (Mobile-First)

스크롤이 긴 데이터 리스트 환경에서도 사용자가 길을 잃지 않도록 상단 메뉴를 고정하고, 상태에 따라 버튼이 동적으로 변하는 UX를 구현했습니다.

## 🛠 주요 기능

* **전문가 프로필 CRUD**: 신규 등록, 상세 조회, 기존 정보 수정
* **사진 아카이빙**: 구글 드라이브 폴더와 연동된 이미지 자동 업로드 및 관리
* **인증 시스템**: 간단한 패스워드 로직을 통한 비인가 사용자 접근 제어
* **실시간 검색**: 이름 및 소속 기반의 동적 필터링 기능

## 📖 시작 가이드 (Quick Start)

1. **시트 및 폴더 준비**
* 데이터 저장용 구글 시트를 생성합니다. (시트명: `시트1`)
* 사진 저장용 구글 드라이브 폴더를 생성하고 공유 권한을 **[링크가 있는 모든 사용자 - 뷰어]**로 설정합니다.


2. **프로젝트 설정**
* `Code.gs` 상단의 `PHOTO_FOLDER_ID`에 본인의 드라이브 폴더 ID를 입력합니다.
* `index.html` 상단의 `PW` 변수에 사용할 접속 비밀번호를 입력합니다.


3. **권한 승인 및 배포**
* GAS 에디터에서 [배포] -> [새 배포] -> [웹 앱]을 선택합니다.
* 액세스 권한을 **[모든 사용자]**로 설정하여 배포 후 생성된 URL로 접속합니다.

## ⚠️ 자주 발생하는 문제 해결 (Troubleshooting)

| 현상 | 원인 | 해결 전략 |
| --- | --- | --- |
| **사진이 보이지 않음** | 구글 보안 정책 차단 | 서버의 **Base64 인코딩 전송** 로직 확인 |
| **Permission Denied** | OAuth 권한 미승인 | 에디터에서 함수 실행 후 **권한 승인** 재진행 |
| **수정사항 반영 안 됨** | 이전 버전 웹 앱 접속 | 배포 관리에서 **새 버전 발행** 확인 |
